01 项目背景

👷 客户简介

xx结构有限公司是一家专业钢结构制造企业，主要从事工业厂房、桥梁、高层建筑等钢结构产品的设计、制造与安装。随着企业规模扩大和业务增长，现有的质量管理方式已无法满足精细化管理需求。

💡 需求来源
  • 基于公司质量管理体系要求
  • 结合实际生产流程特点
  • 各部门（质检部、生产部、信息部等）联合提出
  • 现有手工/Excel管理方式效率低下
🎯 项目目标
  • 实现从原材料入库到成品出厂的全流程质量追溯
  • 提高质量检验工作效率，减少人为误差
  • 实时监控生产过程质量状态
  • 提供准确的质量统计分析数据，支持决策
关键背景：企业目前质量数据主要以纸质记录和Excel表格形式存储，缺乏系统化的质量信息管理平台。随着产量增长，数据分散、追溯困难、统计分析滞后等问题日益突出，亟需建立一套全面、高效的质量管理信息系统（QMS）。

02 现状痛点与建设目标

当前痛点

数据分散难追溯

质量数据分散在纸质记录、Excel、不同部门电脑中，无法实现快速追溯。出现质量问题时，需要人工翻阅大量纸质档案，耗时耗力。

手工录入易出错

检验数据全靠手工记录和转录，存在抄错、漏记、数据不一致等人为误差风险，且无法保证实时性。

统计分析滞后

月度/季度质量报表需要人工汇总，通常T+N才能出具，管理层无法及时获取质量状况，错失最佳改进时机。

不合格品闭环难

不合格品处理流程缺乏系统跟踪，返工/返修后复检记录容易遗漏，处理进度不透明，闭环管理困难。

建设目标

全流程数字化

覆盖原材料检验、生产过程检验（切割→焊接→组装→涂装）、成品检验的全流程数字化管理，实现数据一次录入、全程共享。

一键追溯

通过批次号或产品编号，秒级查询从原材料到成品的完整质量档案，包括所有检验记录、不合格处理、合格证信息。

实时统计分析

自动生成合格率、不良率、趋势分析等图表，支持按工序/班组/时间段等多维度统计，支持自定义报表导出。

二维码合格证

成品自动生成附带二维码的合格证，扫码即可查看完整质量信息，提升客户信任度和产品附加值。

04 原材料检验管理

覆盖原材料从到货登记到合格入库的全流程，确保只有检验合格的原材料才能投入生产。

📋 批次信息管理
  • 登记批次编号、材质类型（Q235B/Q355B/304/316等）、规格型号、供应商、数量、到货日期
  • 系统自动生成批次状态：待检验
  • 支持Excel批量导入批次信息
  • 仅允许删除未发生检验记录的批次
📎 材质证明文件
  • 支持PDF/图片格式上传，每个批次可上传多个文件
  • 文件与批次关联存储，支持在线预览和下载
  • 有权限用户可删除错误上传的文件
👁 外观检验
  • 选择批次，记录检验结果（合格/不合格）、缺陷描述
  • 记录检验员、检验日期
  • 未归档前可修改或删除记录
📊 理化性能测试
  • 录入化学成分：C、Si、Mn、P、S等
  • 录入力学性能：屈服强度、抗拉强度、延伸率
  • 支持Excel模板批量导入测试数据，自动匹配批次
状态流转逻辑：外观检验合格 → 取样送理化测试 → 理化性能合格 → 批次状态更新为"合格" → 允许投入生产。任一环节不合格 → 标记为"待处理" → 触发不合格品登记流程。

05 生产过程检验

覆盖切割→焊接→组装→涂装四大关键工序，实现工序级质量控制和不合格品即时拦截。

✂ 切割工序检验
  • 尺寸精度检测
  • 切割质量：粗糙度、垂直度
  • 支持上传切割质量照片
🔧 焊接工序检验
  • 焊缝外观：咬边、气孔检查
  • 焊脚尺寸测量
  • 无损检测（UT/MT/PT）结果录入及报告上传
📐 组装工序检验
  • 组装精度：孔距、平行度
  • 螺栓扭矩检测值记录
🎨 涂装工序检验
  • 涂层厚度：多点测量平均值
  • 附着力测试结果
⚙ 工单管理与任务分配

工单来源：从ERP同步或手动录入，支持按工单号、产品编号、工序、状态查询

任务分配：生产管理人员可将工单检验任务指派给指定质检员

待检提醒：按工序、班组、优先级展示待检验工单，支持红点/站内信提醒

数据导入：支持按工单+工序导入Excel检验数据（批量录入或设备自动采集）

关键机制：任一工序检验不合格 → 立即触发不合格品登记流程 → 阻断后续工序流转 → 处理完成后复检合格方可继续。工序顺序可配置，检验项支持自定义扩展。

06 成品检验与合格证

所有工序完成后进入成品检验环节，通过综合判定后生成附带二维码的合格证，实现产品身份数字化。

📤 成品报检

工单全部工序完成后，由生产部门发起成品报检，生成唯一报检单号。支持按产品编号、工单号、报检日期、检验状态查询报检列表。

👁 成品外观检验

记录成品表面质量、外观缺陷等详细信息，支持上传缺陷照片。

📏 尺寸精度检测

按图纸要求录入关键尺寸实测值，系统自动比对标准值和公差范围。

🔭 无损检测复核

对成品进行UT/MT/PT检测复核，记录检测结果、检测区域、检测人员、报告编号。

✅ 综合判定

外观、尺寸、无损检测全部合格后，系统自动判定成品"合格"；任一不合格则判定为"不合格"并转入不合格品管理流程。

📲 合格证与二维码

自动生成合格证（产品编号、规格、检验标准、检验结论、质检章），支持打印纸质版或下载PDF。二维码包含产品编号、批次号、合格证编号、主要检验数据URL。

07 不合格品管理

建立从发现不合格到最终关闭的完整闭环管理，支持原因分析、多级审批、处理跟踪和复检验证。

📝 不合格品登记

关联对象：工单或原材料批次

缺陷信息：缺陷类型（表面/尺寸/焊接/涂装/材质）、位置、严重程度（轻微/严重/致命）

附件：支持上传缺陷照片、检测报告

查询：按产品编号/批次号、缺陷类型、处理方式、处理状态等组合查询

🔬 原因分析

记录人机料法环分析结果，选择主要原因类别，为质量改进提供数据支撑。

🛠 处理方式

支持返工、返修、报废三种方式。返工/返修需主管审批，报废需质量经理审批。

🔄 复检与关闭

返工/返修后强制复检，记录复检结论。合格则关闭不合格记录，不合格则升级处理/重新评审。

📈 统计与报告

支持按时间范围/产品类型导出不合格品统计及分析报告（Excel），报废自动统计损失金额，生成不合格品分析报告。

08 质量追溯与统计分析

质量追溯

🔍 按批次号追溯

输入原材料批次号，查询该批次用于哪些工单、各工序检验记录、最终成品信息。支持展示上下游依赖关系。

🔬 按产品编号追溯

输入成品产品编号，展示从原材料到成品的全流程检验记录，以时间轴或流程图形式呈现。

⚠ 异常快速定位

在追溯图中高亮显示不合格工序，展示缺陷详情，快速定位问题环节。

📣 召回管理

选择产品范围（批次号/日期范围），填写召回原因，生成召回通知单，导出召回产品清单及处理结果。

质量统计分析

📈 合格率统计

按工序（切割/焊接/组装/涂装）或班组统计合格率，生成图表+表格对比，支持时间段筛选。

📉 不良率分析

饼图/柱状图展示各类缺陷（表面/尺寸/焊接/涂装/材质）占比。支持按产线/车间横向对比不良率。

📈 趋势分析

按月/周展示合格率变化折线图，支持预测线。可设置质量预警阈值，低于设定值时系统自动发出预警通知。

📄 自定义报表

用户可选择字段、统计维度、筛选条件生成个性化报表，保存为模板。支持导出Excel/PDF。

09 系统管理与非功能需求

系统管理

用户权限管理
  • 用户增删改查，分配角色
  • 角色管理：分配菜单权限、操作权限
  • 按钮级权限控制（如仅允许查看，不允许导出）

  • 操作日志：按用户、时间、模块、操作类型查询，导出Excel审计

● 不合格品闭环：登记 → 原因分析 → 返工/返修/报废 → 复验
数据流向：原材料合格 → 生产工序；工序合格 → 成品检验；复验合格 → 成品检验；所有数据 → 质量追溯数据库
关键点：需支持跨表关联查询；展示上下游依赖关系（如某批次原材料用于哪些产品）；召回管理支持受影响产品批次自动锁定。

15 待确认问题：售前核心问题

以下问题直接影响系统架构设计、实施难度和报价评估，需在需求澄清会上重点确认。

1 客户真实业务复杂度 vs 文档描述的差异
  • 文档中提到的"工序检验"包括切割、焊接、组装、涂装，但未说明是否存在跨车间协同、外协加工、返工流程
  • 是否已有MES、ERP、条码/RFID等系统？若没有，质检数据采集方式（手工录入/PDA/平板）将极大影响系统设计
  • 质检人员年龄结构、IT技能水平如何？是否愿意使用移动端录入？
2 系统集成能力与接口成熟度
  • ERP品牌及版本？是否有开放API？是否可提供接口文档？
  • 生产设备是否具备数据采集接口（如PLC、OPC UA、Modbus）？
  • 数智平台是客户自研还是第三方？接口标准是什么？
  • 是否需要与现有工业互联网平台对接？
3 数据迁移与历史数据清理
  • 现有质量数据是否在Excel、纸质记录中？量有多大？
  • 是否有历史不合格品记录需要迁移？
  • 是否需要与原有系统并行运行（过渡期）？
4 部署方式与安全合规
  • 本地服务器、私有云还是SaaS公有云？
  • 是否需要等保测评（二级/三级）？
  • 是否需要对接企业AD域/单点登录？
  • 是否要求满足ISO 9001审核追溯要求？
